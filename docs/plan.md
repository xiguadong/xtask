# 计划列表

## 近期（1-2 周）

1. 稳定测试基线
2. 补齐 P7 验收矩阵缺失场景
3. 明确 CI 触发策略与失败阻断策略
4. 建立 `docs/task.md` 与 `.agents/task_graph.yaml` 的自动同步脚本（当前已完成首次手动同步）

## 中期（2-4 周）

1. 前端单元测试体系（Vitest + Testing Library）
2. E2E 数据隔离与测试夹具
3. 页面可访问性自动检查（a11y）

## 长期（>4 周）

1. 测试报告可视化与历史趋势追踪
2. 覆盖率门禁（后端 + 前端）
3. 发布流程标准化（版本、变更日志、回归清单）

## 每次迭代固定动作

1. 更新 `docs/task.md` 勾选状态
2. 更新 `docs/plan.md` 阶段目标
3. 运行 `make test && make e2e-test`
4. 更新 `PROGRESS.md`
