# xtask 初始化 milestone 预设

下面是初始化 xtask 时常用的 milestone 建议。默认先给 3 个，避免一开始拆得过细。

## 通用软件项目

1. **仓库接管与基线建立**
   - 适用：首次接入 xtask、需要先把仓库纳入可管理状态
   - 内容：初始化 xtask、注册项目、梳理目录、建立首批任务规则

2. **核心流程打通 / MVP**
   - 适用：需要尽快形成最小可交付版本
   - 内容：聚焦最核心的 1~2 条业务链路或功能闭环

3. **稳定化与首次交付**
   - 适用：已有基本能力，需要收口并准备对外或对内交付
   - 内容：修复关键问题、补验证、整理文档、准备发布

## 遗留项目接入 xtask

1. **现状盘点与数据迁移**
   - 内容：检查旧 `.xtask/`、迁移到 `refs/xtask-data`、确认历史任务可读

2. **任务体系整理**
   - 内容：统一 milestone、清理重复任务、明确 worktree 规则

3. **首轮迭代落地**
   - 内容：基于新的任务体系推进第一批实际开发项

## Web / 前后端项目

1. **基础设施与运行链路**
   - 内容：启动、构建、环境配置、路由/接口基线

2. **核心业务闭环**
   - 内容：完成关键页面、关键接口、主流程打通

3. **联调与发布准备**
   - 内容：联调、修复、验收、上线准备

## CLI / 工具型项目

1. **命令骨架与数据模型**
2. **核心命令可用**
3. **验证、文档与发布**

## 提问后如何选

- 用户强调“先接管现状”时，优先选“仓库接管 / 现状盘点”开头
- 用户强调“尽快可用”时，第二个 milestone 优先叫 “MVP” 或 “核心流程打通”
- 用户强调“马上上线/交付”时，第三个 milestone 明确写“稳定化 / 发布准备”
